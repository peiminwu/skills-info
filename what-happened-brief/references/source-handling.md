# Source Handling

Use this reference file only when choosing sources, checking source quality, or deciding whether a claim is strong enough to include.

The current hierarchy is derived from the maintained source list at:
[source-list](https://raw.githubusercontent.com/peiminwu/source-list/refs/heads/main/list)

## Fact Versus View Rules

Apply these defaults:

1. Facts: prefer level 1, then level 3 for rapid reporting.
2. Views: prefer level 2, named investors, and level 5.
3. Platform commentary: use only as a supplement, and prefer the version with the widest circulation or highest visibility.

## Named-Person Rule

If you cite or paraphrase what a person said:

1. Find at least two independent sources.
2. Make sure both describe the same claim, not different summaries.
3. Do not upgrade a single interview clip into a market consensus.

## Cutoff Rule

For any explicit historical cutoff:

1. Every cited item must have been publicly available on or before the cutoff date.
2. If the publication date is unclear, exclude it.
3. Do not use retrospective analysis published later, even if it discusses earlier events.

## When To Mark Evidence As Weak

Mark `证据状态：一般` or `不足` when:

- The causal story is still debated
- Reporting is mostly second-hand
- You cannot confirm the original statement cleanly
- The source mix leans too heavily on lower-tier commentary
- The outlook section is mostly speculative rather than evidence-backed
